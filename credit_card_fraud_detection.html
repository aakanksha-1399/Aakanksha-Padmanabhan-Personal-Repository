Credit Card Fraud Detection with Real-Time Monitoring

Python Pandas NumPy Scikit-Learn Random Forest Decision Tree Logistic Regression Neural Networks Undersampling Folium Streamlit

Project Overview

This project tackles the challenge of identifying fraudulent credit card transactions using machine learning. With rising digital payments, preventing fraud is critical — this model aims to detect anomalies that signal suspicious activity.

Data Overview

The dataset comprises real-world financial transactions with a significant imbalance between fraudulent and non-fraudulent cases. Missing values and inconsistencies were handled using statistical imputation and data cleaning methods to achieve 100% data completeness.

Categorical variables such as transaction type and location were converted into numerical formats via one-hot and label encoding. Robust scaling techniques normalized transaction amounts and mitigated the impact of outliers.

Visualization showing data distribution and cleaning process

Balancing Dataset

The dataset contains 1.85 million transactions, with only 0.41% classified as fraudulent, creating a significant class imbalance. To address this, three techniques were tested: no balancing, SMOTE oversampling, and undersampling with random sampling. Without balancing, the model was biased and missed many fraud cases. SMOTE improved minority representation but lowered precision and recall. Undersampling balanced the classes effectively, yielding the best precision and recall, and was therefore selected. Model Selection

We evaluated four models in this study: Logistic Regression, Decision Tree, Random Forest, and Neural Network — each selected for its ability to handle complex classification tasks.

  • Logistic Regression: A linear model suitable for binary classification tasks.
  • Decision Tree: Handles categorical and non-linear data, making it effective for identifying complex fraud patterns.
  • Neural Network: A deep learning model that detects subtle patterns across multiple features.
  • Random Forest: An ensemble of decision trees that improves pattern recognition and reduces overfitting.

Evaluation Metrics:

The performance of each model was evaluated using four major criteria:

  • Accuracy: The percentage of total transactions correctly classified as either fraudulent or non-fraudulent.
  • Precision: The proportion of predicted fraud cases that were actually fraudulent — higher precision means fewer false alarms.
  • Recall: The proportion of actual fraud cases that were correctly identified — higher recall means fewer missed frauds.
  • F1-score: The harmonic mean of precision and recall, providing a balanced measure of the model’s fraud detection performance.
