Airline Delay Analysis

Python PostgreSQL MongoDB SQL Azure Data Cleaning Data Normalization Visualization

Project Overview

Analyzed over 60 million flight records across 11 years to uncover patterns in airline delays and cancellations caused by weather, air traffic, and operational issues. Designed efficient relational and document-based data models using PostgreSQL on Azure and MongoDB to handle complex, large-scale datasets. Extracted actionable insights into route performance and delay causes to support smarter scheduling and reduce flight disruptions.

Data Overview

The dataset spans 11 years of airline flight data across the US and includes over 60 million records. It captures flight delays, cancellations, and causes ranging from weather and carrier issues to NAS and security. Data was sourced from Kaggle, requiring significant preprocessing due to size, inconsistencies, and formatting challenges.

Each flight record contains over 25 attributes, including timestamps, airport codes, airline identifiers, and delay durations. The dataset was loaded into PostgreSQL and MongoDB for different phases of analysis. Extensive cleaning was performed, including type conversions, null handling, timestamp standardization, and relational splitting.

Data Modeling

The data was structured using both relational and document-oriented models. In PostgreSQL, flights were normalized across multiple tables:

  • Flight
  • FlightInformation
  • CancelledFlights
  • DelayCausation
  • Delayed

following 3NF principles. Primary keys and foreign key constraints ensured data integrity and optimized joins.


An alternative MongoDB model was also created, nesting flight metadata, delay types, and status inside a single document. This allowed flexible querying across years with varying schema completeness. Indexes were added on carrier and origin-destination fields to improve performance. The ER diagram helped define relationships, and a document schema was used to demonstrate NoSQL flexibility.

Data Cleaning

Accurate airline delay analysis relies heavily on clean and reliable data. To ensure this, we undertook the following steps:

  • Duplicate Removal: Detected and removed duplicate flight records identified by Flight ID, preventing inflated delay counts.
  • Canceled Flights Filtering: Excluded canceled flights missing cancellation codes, since these entries do not contribute to delay insights.
  • Delay Data Correction: Cross-checked departure delay values against scheduled and actual departure times to identify and fix inconsistencies.
  • Invalid Timestamp Exclusion:Removed flights with unrealistic time entries, such as departures occurring before the scheduled time, to maintain data accuracy.

Azure Backup Server

To ensure data security and reliability, an Azure subscription was used to set up backup servers. This provided automated, scalable cloud backups that safeguard our datasets against loss or corruption, enabling seamless recovery and maintaining continuous availability.

Results

Our analysis revealed that the National Aviation System (NAS) delays are the primary cause of flight disruptions, accounting for the majority of delays in 2018. Southwest Airlines experienced the highest number of delays, while a few carriers showed significant flight cancellations in 2015. We also identified key airline collaborations and destination pairs that peak during holiday seasons, suggesting opportunities for operational improvements and package deals. These insights offer valuable guidance for airlines and airport authorities to prioritize resources, reduce delays, and enhance passenger satisfaction.
